工作经历

阿里旅行（2015年8月 ~ 至今）

负责支付系统的开发设计，利用 Selenium 开发模拟支付功能，搭建全新基于 windows 服务器的应用，设计 win 部署发布流程，解决 HSF 、Notify、KeyCenter 等中间件遇到的各种问题 ；利用 Webx3 、Bootstrap 、jQuery 及 相关 Plugin 搭建代理商后台管理系统，开发订单列表、详情、支付账号管理、代理商信息管理等页面；
作为 PM ，整理协调项目的需求、开发、测试进度，解决项目中遇到的问题，保证项目正常上线。

交易系统

开发交易系统迭代需求，不断优化完善流程。加班 1 月搞定火车票售卖自主保险，对接保险平台，重构交易链路流程，项目上线后使得火车票的盈利成为可能；出票锁单、随机立减优惠活动、多保险等项目通过与其他团队合作，优化购票流程，提升用户体验；适应前端、平台、交易中心、优惠中心、支付宝等与其他团队协同作战的开发模式，提高自己沟通表达能力。
熟练使用 HSF 、Notify 、TMC、TC 消息中心等组件，利用 Python 脚本处理 1300+ 服务器代理变更为 Squid ，调整访问策略，控制白名单，自动验证机器可用性等，减少增加服务器的成本。

代理商小号平台

独立申请新应用，根据团队任务职责规范搭建应用框架，开发基于阿里小号的代理商通信平台，方便代理商使用，节省其成本。
作为加入团队的第一个项目，采用 Bootstrap 绘制后台管理系统页面，熟悉 Webx 3 的开发流程，熟悉新应用的申请、Aone2 打包部署分支开发上线流程，熟悉团队任务职责链开发框架，技术中间件 Notify、TMC 消息队列的使用，集团 RPC 通信框架 HSF 的使用，集团 TC 短信中间件的使用等。
平台高峰期日发送短信 60w 条，节省代理商大量成本，通过雷达实时日志监控，保障系统可用性。分享 Notify、TMC 问题排查方法，提升大家工作效率。

艺龙网 （2012年7月 ~ 2015年7月）

海量日志存储

8 周时间 1 个人从需求收集、架构设计、程序开发、打包部署到线上环境运维，搭建了一套分布式实时日志收集存储系统，基于开源的 Apache Flume、ZooKeeper、Kafka、JStorm、HBase、Hadoop、Phoenix、百度 Echarts 图表组件，解决 PC 网站端各个频道线上日志采样收集、数据延迟的问题，实时展示线上异常日志，汇总展示每小时 Top 异常报表。每日收集日志 1500w 条，查看线上日志实时性降低到秒级（原 15-30 分钟），可追踪数据延长到 1 个月（原 7 天）。

汽车票项目

负责艺龙网网站端首版汽车票列表页、订单填写页、订单完成页三个核心页面的开发：引入 jQuery template 开源组件，前后端交互只传递 JSON 数据；利用 FreeMarker list 特性，开发通用分页组件；利用百度地图 API，定制汽车票频道专有地图页；利用 Hudson+Maven 插件开发 Java 项目的 CI 部分。

通过 AJAX 异步，合并 CSS、JS、图片，减少 HTTP 请求，设置缓存时间等优化线上性能，列表页 TPS 为 44，平均响应时间 500ms，订单填写页 TPS 为 110，平均响应时间 200ms，相比之前 25 左右的 TPS，性能提升 76%~300%。

EFC 分布式文件存储系统

基于开源组件 FastDFS 搭建的轻量级分布式海量文件存储服务，网站、H5、无线 App 均在使用，每日存储量 1~10GB 数据。

负责存储端 MySQL 表设计；管理端基于目录的权限设计开发，文件上传、发布、删除、回滚操作的开发；首次使用 Git 版本管理，负责 Git 版本控制下的打包部署；基于 Semantic UI 前端框架设计整个管理端布局样式；设计首批 RESTful 协议开放 API 接口规范；

涉及技术：Java Spring MVC 框架，MyBaits 数据持久层框架，Maven 依赖打包管理，前端 Semantic UI 语义化框架，jQuery loadtemplate 开源数据绑定模板，WebUploader 开源上传组件，FastDFS 分布式存储，Git 版本管理。

最自豪的事： 开始学习使用 Git，完成项目的 Git 的 CI 过程，整理出一套合适的分支开发工作流程，在部门团队开始推广 Git，组织几次分享，内容已经分享到 GitHub，star 数 2400+，fork数 900+；（目前团队已经开始全面使用 Git 版本控制）

搜索V4版重构

参与酒店搜索系统的重构，负责 list 搜索静态模块、all in one 智能搜索的整合、rank 模块的设计及开发，V4 整体性能提升 70%，tps 提升 10 倍，扩展到支持 C#、Java、C++ 等多种语言客户端。
涉及技术：系统模块分层、C# Linq Lamda 表达式，C# 多线程处理，缓存 Cache 模块设计 Memcached 使用，RPC 框架 Thrift 的使用，rank 策略模块设计

挑战：整个酒店搜索业务复杂，代码业务性太强，理解起来费劲，涉及到 C# Lamda 的一些高级用法看不懂，项目时间短，身为 Scrum Master，必须保证项目进度与质量，面临考验巨大；通过不断向老前辈沟通请教业务逻辑，研读 C# 语言相关的内容，请教团队老人，做笔记整理，不断学习理解，和测试反复验证功能可用性，向其它 Master 学习经验，研读相关书籍，不断与团队成员沟通，最终完成整个项目的重构，V4搜索重构团队获得当季度公司最佳团队。

AIO智能搜索

酒店关键字搜索系统，根据用户输入 Query 返回符合的酒店列表。
负责优化当前系统，包括基于位置附近酒店功能模块、新增多粒度分词及分词词库的增量更新机制，来解决解决酒店名全文检索问题；加入 POI+酒店名全搜混排，POI 搜索 5 公里、10 公里分段召回酒店的功能，来解决 psg 对于商圈搜索酒店排序与普通用户不一致的问题。
挑战：由于原系统负责人离职，对整个系统及搜索概念不熟悉，压力很大，通过不断研读源码，调试跟踪，整理笔记，买书籍阅读了解搜索相关的技术，包括分词、倒排、全文检索等，完成了业务的需求。

库存分布式存储迁移

库存 SQL Server 单表数据达到 1.6 亿，为应对高峰期压力，团队决定经过垂直拆分库存库、水平迁移分布式两步将库存库迁移到分布式 MySQL 集群（16shard），降低 bdg 主库压力，提高库存维度系统的容量，同时也为酒店组后续订单库拆分、迁移留下宝贵的经验。
期间负责协调其它团队收缩库存的访问、MySQL 语句的优化、数据迁移工具开发、相应业务功能开发等。
挑战：开始了解分布式，熟悉 MySQL 语句的优化，在短时间需要完成较多的工作，团队整体加班奋斗一个多月保证项目上线。

酒店产品重构维护

参与酒店产品团队需求的开发，包括多供应商系统构建，使产品多元化，提升竞争力，增加酒店间夜量；库存预测系统搭建，实现未来一个月内酒店的库存、价格与竞争对手的对比，方便 psg 争取更优惠的价格和更多的库存，提高用户在 elong 网上的转化率；酒店结算系统重构，增加多币种支持等。
涉及技术：Html，jQuery，JavaScript，CSS，C#，.NET Mvc，WCF，Linq Lamda 表达式，MessageQueue 等。
领导评语：进入公司后，快速融入团队，熟悉开发过程，成为团队开发主力，高质量的完成开发任务，超出预期。
