## 二、项目愿景

父亲 xirong 的终极目标是**个性化教学** — 借助 AI 能力，为每个孩子量身定制学习内容。具体实践方式是：

1. **从真实兴趣出发**：父子一起在 B 站看太阳系科普视频（BV1Wd4y1r7UE，共 15 集），孩子对宇宙产生浓厚兴趣
2. **视频知识转化为互动程序**：看完每集后，父亲告诉 Claude 这集讲了什么知识点，Claude 自动生成互动闯关页面
3. **跨学科融合学习**：在天文科普中融入汉字学习（每章 2 个字）和 10 以内数学（每章 1 道题），一举三得
4. **边玩边学**：所有内容以 3D 交互 + 闯关游戏的形式呈现，不是传统的题库练习

每个星球包含 3 个核心文件：

| 文件 | 用途 | 平均行数 |
|------|------|---------|

#### 汉字教学设计

8 个星球共教授 **96 个汉字**（每星球 12 字），全部与该星球主题关联。每个汉字的教学流程：

```
大字展示（字号 120px）
    ↓
拼音标注（如 yuè）
    ↓
组词（月亮 · 月球）
    ↓
造句（月亮挂在天上，又大又圆。）
    ↓
Canvas 象形图动画（如"月"字画一弯月亮，从象形演变到楷书）
    ↓
语音朗读（Edge-TTS 高质量中文女声）
```

象形图绘制使用 Canvas 2D API 手工编码，每个汉字都有独立的 `drawXxx()` 函数，用线条和色块表现从甲骨文象形到现代汉字的演变。全项目共有 **100+ 个象形图绘制函数**。

#### 答案位置规范

为防止孩子记住"答案总在 B"的规律，制定了严格的答案分布规则：

```
         第1章  第2章  第3章  第4章  第5章  第6章
知识测验:   A     C     B     A     C     B
数学挑战:   B     A     C     B     A     C
```

规则：相邻章节位置不同、同章内测验和数学位置不同。

```
└── starfield.jpg         # 星空背景
```

#### 渲染架构：纹理 + Shader 混合

```
真实纹理（基础层）— NASA/Solar System Scope 卫星照片
    ↓ texture2D() 采样
自定义 ShaderMaterial（效果层）
    ├── 昼夜光照切换（sunDirection uniform）
    ├── Fresnel 大气散射边缘发光（每个天体不同颜色和强度）
    ├── 云层（地球独立球体，radius * 1.02）
    └── 动态自转（time uniform）
```

**核心原则**：凡是渲染现实世界中真实存在的物体，永远不要用纯代码去"画"它——用真实照片纹理贴图，再用 Shader 叠加光照和交互效果。app.js 中有 52 处 ShaderMaterial 相关代码。

由于是纯静态网站（无后端），语音方案采用**预生成 MP3 + 运行时降级**：

```
开发阶段（本地 Python）                   运行时（浏览器）
┌──────────────────────┐              ┌──────────────────────────┐
│  edge-tts 库          │              │  playAudio(path, text)   │
│  ↓                    │              │    ↓                     │
│  zh-CN-XiaoxiaoNeural │  → MP3 →    │  new Audio(path).play()  │
│  （小晓 - 儿童友好女声）│              │    ↓ 失败时               │
│  ↓                    │              │  speak(text)             │
│  audio/{planet}/*.mp3  │              │  （Web Speech API 备选）  │
```

## 六、开发方式：AI 协作的工程实践

### 6.1 与 Claude Code 的协作模式

这个项目**全程使用 Claude Code（Anthropic 官方 CLI）协作开发**，形成了一套独特的工作流：

1. **父亲提需求**：用语音输入法口述（"给智天做一个火星探险家页面，6 章闯关"）
2. **Claude 研究代码库**：自动扫描 CLAUDE.md、已有代码结构、纹理资源
3. **Claude 生成完整代码**：HTML + JS + Python 音频脚本，遵循项目规范
4. **自动化验证**：Claude 启动本地服务器 → Playwright 打开浏览器 → 截图/快照验证 → 零错误确认
5. **自动提交部署**：git add → commit → push → GitHub Pages 自动部署

### 6.2 多智能体并行开发

星球探险家系列的 7 个星球（除月球外）是通过 **Claude Code 多智能体团队**一次性并行生成的：

```
主 Agent（团队领导）
├── sun-agent    (port 8770) → 太阳探险家
├── mars-agent   (port 8771) → 火星探险家
├── jupiter-agent(port 8772) → 木星探险家
├── saturn-agent (port 8773) → 土星探险家
├── mercury-agent(port 8774) → 水星探险家
├── venus-agent  (port 8775) → 金星探险家
└── neptune-agent(port 8776) → 海王星探险家
```

- 分两批执行（4 + 3），避免资源竞争
- 每个 Agent 独立完成：代码编写 → 音频生成 → Playwright 自测
- 主 Agent 负责最终的 index.html 整合和统一提交
- 单次提交 **523 个文件，19,184 行新增代码**

## 九、项目特色与亮点

1. **纯前端零依赖部署**：全部代码在 GitHub Pages 运行，无需后端、数据库或 API 密钥，任何设备打开浏览器即可使用
2. **真实纹理渲染**：所有天体使用 NASA/Solar System Scope 卫星照片，不是卡通风格，让孩子看到真实的宇宙
3. **跨学科融合**：一个页面同时教天文、汉字、数学三个学科，基于同一个主题串联
4. **AI 原生开发**：从需求到代码到测试到部署，全流程由 Claude Code 完成，单次最大规模提交 523 文件
5. **高质量中文语音**：780 个 MP3 使用微软 Edge-TTS 小晓声音预生成，接近真人发音
6. **儿童友好交互**：大字体、大按钮、闯关激励（星星 + 徽章）、进度保存、答错可重试
7. **来自真实生活**：高铁模拟基于父子真实乘车经历，天文内容来自一起看的 B 站视频
